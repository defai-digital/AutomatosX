## Technical Details

### Why Warnings Occur
- AutomatosX: tree-sitter v0.25.0 (latest)
- 40+ language packages: expect v0.21.x / v0.22.x
- Parsers work correctly with v0.25.0
- Only metadata outdated

### Why pnpm.peerDependencyRules Doesn't Help End Users
- Configuration in package.json applies during **development**
- npm publish doesn't include these rules in **package metadata**
- End users' pnpm reads peer deps from **upstream packages**
- Upstream packages declare incompatible peer dep ranges
- **Result**: Warnings appear for end users

### Why Documentation Is the Right Fix
‚úÖ No code changes (zero risk)
‚úÖ User choice (multiple options)
‚úÖ Works with npm and pnpm
‚úÖ Fast to implement (30 min)
‚úÖ Transparent about issue
‚úÖ Doesn't compromise dependency management

**Rejected alternatives**:
‚ùå Bundle dependencies (3x package size)
‚ùå Update upstream packages (out of our control, months/years)
‚ùå peerDependenciesMeta (doesn't apply to transitive deps)
