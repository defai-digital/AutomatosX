# 競品比較表（LangChain / AutoGen / CrewAI）

| 維度 | AutomatosX | LangChain | AutoGen | CrewAI |
|---|---|---|---|---|
| 核心定位 | 可靠性與治理的 AI coding 平台 | LLM chain 與工具編排 | 多 agent 對話協作 | 角色分工式多 agent 工作流 |
| 治理與合規 | 內建 Guard + 合約不變性 | 需自行補強 | 需自行補強 | 需自行補強 |
| 可審計性 | Trace + 事件化記憶可回放 | 依整合程度 | 依整合程度 | 依整合程度 |
| 架構分層 | 合約優先，核心與供應商解耦 | 可擴充但易耦合 | 對話編排為主 | 角色編排為主 |
| 多模型策略 | 路由引擎，基於風險/能力選擇 | 可自行實作 | 可自行實作 | 可自行實作 |
| 使用門檻 | 較高（工程化程度高） | 低到中 | 中 | 低到中 |
| 適用場景 | 企業級、可持續維運 | 快速 prototype | 多 agent 協作研究 | 團隊式任務拆解 |

## 總結差異化
- AutomatosX 重點在「可靠性、治理、可審計」，不是單純 agent 編排。
- LangChain / AutoGen / CrewAI 偏「快速驗證與流程編排」，工程治理較薄弱。
