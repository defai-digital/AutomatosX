# 商業化路線與產品分層（開源 / 企業版）

## 分層策略
**開源版（OSS）**
- 核心合約與核心域：routing / workflow / memory / trace
- 基礎 Guard policy 與 CLI / MCP server
- 標準多模型路由與基本日誌
- 社群插件與範例工作流

**企業版（Enterprise）**
- 進階治理：可視化 policy 管理、合規報告、審計存證
- 安全與權限：SSO / RBAC / 조직級權限
- 企業整合：私有模型、內部工具、VPC 部署
- SLA 與支援：穩定性保證與專家導入服務

## 商業化路線（建議）
**Phase 1：建立信任基礎**
- 強化開源核心，主打「AI coding 可靠性」
- 釋出標準化的治理與審計模組

**Phase 2：企業化治理套件**
- 提供合規與審計擴展
- 支援組織級政策管理與權限控管

**Phase 3：平台化與生態**
- 建立 Marketplace / 模板庫
- 形成多模型與工作流的可重用生態

## 定價與價值溝通
- 將價值聚焦在「降低風險 + 降低治理成本 + 提升可信任度」
- 對企業說清楚：AI coding 的價值不是模型效能，而是系統可靠性
