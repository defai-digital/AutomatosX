### Intent Classification

**IntentClassifier** uses hybrid approach:

1. **Pattern Matching (Fast Path)** - 40+ regex patterns for common intents:
   - Memory search: `find`, `search`, `show`, `list`, `where is`, etc.
   - Workflow execution: `run`, `execute`, `start`, `launch`, etc.
   - Agent delegation: `use`, `ask`, `talk to`, `delegate to`, etc.

2. **LLM Classification (Fallback)** - When patterns don't match:
   - Uses Claude Haiku (fast, cheap model)
   - Returns intent type + confidence + extracted parameters
   - Caches results for performance

3. **Intent Learning** - Improves over time:
   - Tracks user corrections
   - Adjusts confidence thresholds
   - Builds user-specific patterns

### Context Management

**ConversationContext** maintains:
- Conversation ID (UUID)
- User ID
- Session start time
- Message count
- Active agent (optional)
- Active workflow (optional)
- Variables (key-value store)
- Last results (for chaining operations)

**Persistence**:
- Conversations stored in `conversations` table (SQLite)
- Messages stored in `messages` table with FK to conversations
- Context serialized as JSON in `conversations.context` column
- Automatic save on each message
- Restore session with `/load <conversation-id>`
